# 智能长截图插件（Chrome Extension）

一个面向复杂网页文档场景的长截图插件，适配普通页面与“特殊滚动容器”页面（如飞书文档、钉钉文档、知识库页面等），支持自动整页、CDP 增强整页、自定义起止区间截图，方便归档和后续交给大模型做文档分析。

---

## 项目定位

在实际办公场景中，很多文档站点会使用：

- 内部滚动容器（不是 `window.scroll`）
- iframe 嵌套
- 固定侧边栏/悬浮工具条
- 虚拟列表或动态渲染

传统长截图工具在这类页面上常出现：只截一屏、重复拼接、错位、边栏混入等问题。  
本插件的目标是用更稳的截图链路，尽可能得到“可阅读、可归档、可分析”的长图输出。

---

## 核心特色

### 1) 多模式截图

- `截取整个网页`：常规滚动拼接，适合普通网站
- `CDP整页（最完整）`：基于 Chrome DevTools Protocol 的增强模式，适合复杂页面
- `自定义（选起止位置）`：先标记开始，再标记结尾，按区间自动截图拼接

### 2) 针对复杂页面优化

- 识别并处理 iframe / 内部滚动容器
- 分段滚动 + 分帧截图 + 自动拼接
- 自动去重重叠区域，降低重复拼接
- 自动裁剪顶部/底部干扰区与正文外侧边栏（保留正文主列）

### 3) 更适合大模型分析

- 输出连续、结构更完整的文档长图
- 减少重复段落和无关 UI 干扰
- 便于后续 OCR、摘要、对比、要点提取等分析流程

---

## 适用场景

- 飞书文档、钉钉文档、知识库页面归档
- 项目复盘材料截图留存
- 需求评审资料生成“可分享长图”
- 提供给大模型进行内容分析、问答、总结

---

## 权限说明（Manifest）

当前插件主要使用以下权限：

- `activeTab`：操作当前激活页
- `scripting`：注入页面脚本执行拼接逻辑
- `debugger`：调用 CDP 能力做增强截图（复杂页面关键）

---

## 安装方式（开发者模式）

1. 打开 Chrome：`chrome://extensions/`
2. 右上角开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本项目目录
5. 在扩展列表中确认插件加载成功

---

## 使用说明

### A. 普通整页

1. 打开目标页面
2. 点击插件按钮
3. 选择 `截取整个网页`

### B. CDP 增强整页（推荐复杂页面）

1. 打开目标页面
2. 点击插件按钮
3. 选择 `CDP整页（最完整）`
4. 按提示授权调试能力（若浏览器弹窗）

### C. 自定义起止区间

1. 打开目标页面并滚到起始位置
2. 点击插件按钮，选择 `自定义（选起止位置）`
3. 在页面控制面板点击 `设为开始`
4. 滚动到结束位置，点击 `设为结尾并开始`
5. 等待自动拼接并导出图片

---

## 已知限制

- 极端动态页面（持续动画/异步刷新的内容）仍可能带来轻微拼接误差
- 部分跨域 iframe 受浏览器安全边界影响，稳定性依赖页面实现
- 超长页面截图耗时会增加，建议优先使用“自定义区间”减少无关内容
